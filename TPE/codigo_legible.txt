Comandos de git:
	git init Iniciar un repositorio vacío en una carpeta específica
	git status Revisamos el estado de los archivos
	git add ‘nombre_de_archivo’ Añadir un archivo específico al área de preparación
	git add . Añadir todos los archivos del directorio
	git commit –m “mensaje” Confirmar los cambios realizados
	git log Muestra el historial de confirmaciones
	git diff Muestra los cambios con respecto a la última versión guardada en el repositorio
	Clear: Limpia la consola

Branch / Ramas
	git branch [rama]: crea una nueva rama 
	git branch -d: borra la rama 
	git checkout [rama]: cambia el puntero HEAD a [rama]
	git merge [rama]: fusiona el contenido de [rama] en la rama en la que estamos parados

Comandos para GitHub:
	git remote: Ver repositorios remotos configurados 
	git remote add [nombre] [url]: Añadir repositorio remoto 
	git remote rm [nombre]: Remover repositorio remoto 
	git Branch: Ver ramas existentes en el repositorio 
	git push [nombre-remoto] [rama]: Subir confirmaciones a repositorio remoto 
	git pull [nombre-remoto] [rama]: Actualizar de repositorio remoto 
Git cambios:
git commit --amend -m Permite modificar el último commit realizado.
git checkout -- [archivo] Permite revertir archivos individuales o repositorios enteros
git reset --hard deshacer todos los cambios en archivos modificados y preparados. (⚠️ los cambios se descartan por completo).
git reset HEAD <archivo> deshacer el archivo preparado.
git reset HEAD~[N] deshacer los últimos N commits manteniendo las modificaciones.
git reset [HASH] se deshacen los commits posteriores al commit especificado.
git reset –soft los cambios quedan en staging área. 
git reset –mixed (opción default) los cambios no quedan en staging área (deben ser agregados)
git revert <HASH> Permite revertir el estado del proyecto, generando un nuevo commit que revierte los cambios realizados. (Es más seguro que un reset).
git stash Permite descartar los cambios, pero guardar
provisionalmente los archivos modificados.
git stash save [comentario] Permite descartar los cambios, pero guardar
provisionalmente los archivos modificados con comentario.
git stash pop los cambios guardados en el último stash vuelven al working directory.
git stash list se muestra el contenido de la pila de stash.
git stash apply <STASH> se aplica sobre el working directory los cambios del stash especificado.
git stash drop <STASH> elimina de la lista el stash especificado.
